# TQC Java 模擬試題 (Test01)

這是一份模擬 TQC 風格的 Java 試題，旨在測試您的基礎程式設計、邏輯、演算法以及物件導向設計能力。

---

### 第一題：BMI 指數計算與評級

#### 題目說明
請編寫一個程式，提示使用者輸入他們的身高（單位：公尺）和體重（單位：公斤）。程式需計算其身體質量指數（BMI），並根據以下標準印出對應的健康評級。

**BMI 計算公式**: `體重 (kg) / (身高 (m) * 身高 (m))`

**評級標準**:
- BMI < 18.5: "體重過輕 (Underweight)"
- 18.5 <= BMI < 24: "體重正常 (Normal)"
- BMI >= 24: "體重過重 (Overweight)"

#### 要求
1. 程式需包含一個名為 `calculateAndRateBMI` 的靜態方法，該方法接收身高和體重作為參數，並回傳一個包含評級結果的字串。
2. 在 `main` 方法中，呼叫 `calculateAndRateBMI` 方法並印出結果。
3. 需處理不合法的輸入（例如，身高或體重為零或負數），若輸入不合法，應回傳 "輸入無效 (Invalid Input)"。
4. BMI 計算結果保留小數點後兩位。

#### 輸出範例
```
請輸入您的身高（公尺）: 1.75
請輸入您的體重（公斤）: 68
您的 BMI 指數為 22.22，評級為：體重正常 (Normal)
```
```
請輸入您的身高（公尺）: -1.75
請輸入您的體重（公斤）: 68
輸入無效 (Invalid Input)
```

### 第二題：次方計算（迴圈與遞迴）

#### 題目說明
請編寫一個程式，其中包含兩種方法來計算一個整數的非負整數次方 (`base^exponent`)。

#### 要求
1. 編寫一個名為 `powerLoop` 的靜態方法，使用 `for` 或 `while` 迴圈來計算結果。
2. 編寫一個名為 `powerRecursive` 的靜態方法，使用「遞迴」來計算結果。
3. `main` 方法需提示使用者輸入一個基數（`base`）和一個指數（`exponent`，非負整數），然後分別用上述兩種方法計算結果並印出。
4. `powerRecursive` 的基本情況 (Base Case) 應為 `exponent == 0` 時回傳 `1`。

#### 輸出範例
```
請輸入基數: 3
請輸入指數: 4
使用迴圈計算 (3^4) = 81
使用遞迴計算 (3^4) = 81
```

### 第三題：物流包裹費用計算系統 (物件導向設計)

#### 題目說明
您需要為一個物流公司設計一個包裹費用計算系統。系統需要處理標準包裹、快捷包裹和超大包裹三種類型。

#### 要求

1.  **建立一個 `Package` 抽象類別**:
    -   擁有 `protected` 屬性：`trackingNumber` (字串, 追蹤號碼) 和 `weight` (double, 重量 kg)。
    -   擁有一個建構子來初始化這些屬性。
    -   擁有一個 `public abstract double calculateCost()` 抽象方法，用於計算費用。

2.  **建立 `StandardPackage` 類別**:
    -   繼承自 `Package`。
    -   運費計算方式為：`重量 * 3.5` (每公斤 3.5 元)。
    -   實作 `calculateCost()` 方法。

3.  **建立 `ExpressPackage` 類別**:
    -   繼承自 `StandardPackage`。
    -   擁有一筆固定的「優先處理費」12 元。
    -   覆寫 `calculateCost()` 方法，其費用為「標準運費（可透過 `super` 取得） + 優先處理費」。

4.  **建立 `OversizePackage` 類別**:
    -   繼承自 `StandardPackage`。
    -   如果重量超過 20 公斤，需加收一筆「超重附加費」。
    -   覆寫 `calculateCost()` 方法，邏輯如下：
        -   先計算標準運費。
        -   如果重量大於 20 公斤，則最終費用為「標準運費 * 1.5」；否則，費用就是標準運費。

5.  **建立 `ShippingDB` 管理類別**:
    -   內部使用 `HashMap<String, Package>` 來儲存包裹，以追蹤號碼為鍵。
    -   提供 `addPackage(Package p)` 方法來新增包裹。
    *   提供 `calculateTotalRevenue()` 方法，計算並回傳資料庫中所有包裹的總運費。
    *   提供 `findHeaviestPackage()` 方法，找出並回傳最重的那個 `Package` 物件。如果沒有任何包裹，則回傳 `null`。

6.  **建立 `main` 主程式**:
    -   在 `main` 方法中，建立至少三種不同類型的包裹物件。
    -   將這些物件加入 `ShippingDB`。
    -   呼叫 `calculateTotalRevenue()` 和 `findHeaviestPackage()` 方法，並印出總收益和最重包裹的追蹤號碼與重量。

#### 輸出範例 (數值可能因您的實作而異)
```
所有包裹的總收益為: 156.5
最重的包裹是: T003，重量為: 25.0 kg
```
